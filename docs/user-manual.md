# MatchFlow 使用手册

更新日期：2026-06-17

## 1. 启动项目

在项目目录执行：

```bash
npm install
npm run dev
```

浏览器打开 Vite 输出的本地地址，通常是：

```text
http://localhost:5173
```

生产构建验证：

```bash
npm run build
```

构建产物在 `dist` 目录中。当前项目是纯前端应用，部署静态构建产物即可运行。

## 2. 页面结构

系统分为两个主要页签：

- `积分赛程`：配置队伍和场地、生成单循环赛程、安排比赛、录入比分、查看排名对战表。
- `排位赛`：从单循环前 4 名生成排位赛，或手动输入 4 支队伍直接创建排位赛。

页面刷新后，如果系统检测到本地保存的比赛进度，会显示恢复提示。
系统也会记住上次所在页签、赛程表视图和排名排序方式。

## 3. 单循环积分赛

### 3.1 配置队伍和场地

1. 进入 `积分赛程` 页签。
2. 设置 `队伍数`。
3. 设置 `场地数`。
4. 如需自定义队伍名，在队伍名称输入区修改名称。
5. 如需指定球馆实际场地，在场地编号输入区修改名称。
6. 点击 `生成赛程`。

说明：

- 不修改队伍名时，系统默认使用 `队伍 1`、`队伍 2` 等。
- 不修改场地编号时，系统默认使用 `场地 1`、`场地 2` 等。
- 场地编号输入纯数字时，例如输入 `6`，生成后显示为 `6号场地`。
- 也可以输入完整名称，例如 `A馆`、`中心场`。
- 生成赛程后，队伍数、场地数、队伍名称和场地编号会锁定在当前赛程中。
- 如需重新配置，点击 `清空赛程`，并输入确认密码 `789`。

### 3.2 查看概览

生成赛程后，页面会展示概览信息：

- 总比赛。
- 进行中。
- 待比赛。
- 已完成。
- 完成进度。

这些信息会随着比赛安排和结束自动更新。

### 3.3 按计划安排比赛

每个场地卡片会显示该场地的推荐下一场。

默认情况下，推荐来自计划赛程表：

1. 找到空闲场地。
2. 查看推荐对阵。
3. 点击 `安排比赛`。
4. 比赛进入 `进行中` 状态。

如果推荐对阵中的队伍还在其他场地比赛，系统会提示队伍仍在其他场地进行中，并禁用安排按钮。此时可以等待，也可以使用动态调配。

### 3.4 动态调配场地

当某个场地空闲，但计划下一场暂时不能打时：

1. 在该场地卡片点击 `动态调配场地`。
2. 系统为当前场地推荐一场可安排比赛。
3. 确认可接受后点击 `安排比赛`。

动态调配规则：

- 不推荐正在其他场地比赛的队伍。
- 尽量避免刚刚结束比赛的队伍连续比赛。
- 同等条件下优先计划顺序靠前的比赛。
- 动态调配只改变当前场地推荐，不会自动开始比赛。
- 如果没有可调配的补位比赛，页面会显示轻提示。

在总赛程表中，如果实际场地不同于计划场地，会显示 `调至 场地 X`，并使用实际场地颜色提示。

### 3.5 跳过一次

如果当前推荐不合适：

1. 在场地卡片点击 `跳过一次`。
2. 当前推荐会顺延。
3. 继续点击会依次查看后续候选。
4. 所有候选都跳过一轮后，会回到第一个候选。

安排比赛后，该场地的跳过记录会自动清空。
如果没有其他可跳过的候选比赛，页面会显示轻提示。

### 3.6 撤回安排

如果误点了 `安排比赛`，或现场临时需要退回：

1. 在进行中的场地卡片点击 `撤回安排`。
2. 确认后，比赛回到待比赛状态。
3. 场地重新空闲。

注意：

- 撤回只针对进行中的比赛。
- 已经录入比分并结束的比赛不会通过该按钮撤回。

### 3.7 记录比分并结束

比赛完成后：

1. 在对应场地卡片点击 `记录比分并结束`。
2. 输入双方比分。
3. 确认后，比赛状态变为已完成。
4. 场地释放，系统更新概览和排名对战表。

说明：

- 当前版本以胜负为主要统计依据，建议不要录入平分。
- 如果比分相同，系统会阻止保存。
- 结束后比分会参与胜场、净胜分和对战表统计。

## 4. 单循环总赛程表

总赛程表支持两种视图：

- `计划表`：按初始计划轮次和计划场地展示。
- `现场表`：按实际场地和实际顺序展示，已经调场的比赛会出现在实际发生的位置。

两种视图都以矩阵形式展示：

- 行：轮次。
- 列：场地。
- 单元格：该轮该场地的对阵。

轮次生成规则：

- 如果每个传统单循环轮次的比赛数不超过场地数，系统保留传统单循环轮次。
- 如果比赛数超过场地数，系统会把后续可安排比赛提前补入空场地，尽量让场地不空闲。
- 同一个计划轮次内，同一队伍不会被安排重复出场。

颜色说明：

- 表头颜色代表场地颜色。
- 如果某场比赛被动态调配到其他场地，单元格会使用实际场地颜色。
- 单元格内会显示 `调至 场地 X`。
- 如果比赛已有实际场地和现场顺序，会显示对应场地颜色的小圆序号。

点击有比赛的单元格，可以查看比赛详情：

- 计划场地和实际场地。
- 现场顺序。
- 比分。
- 开始时间和结束时间。
- 持续时间。

导出按钮：

- `导出计划表`：始终导出初始计划对阵表，只包含计划轮次、计划场地和对阵。
- `导出当前视图`：导出当前选中的计划表或现场表。

视图记忆：

- `计划表` / `现场表` 的选择会自动保存。
- 单循环赛程表和排位赛赛程表会分别记住各自的视图。

## 5. 排名对战表

排名对战表展示单循环统计结果。

表格包含：

- 排名。
- 队伍。
- 胜场。
- 净胜分。
- 各队之间的比分。

比分单元格含义：

- `-`：自己对自己，或无对应比赛。
- `未赛`：比赛还未开始。
- `进行中`：比赛正在进行。
- `比分`：比赛已完成。
- 胜方和负方使用不同样式区分。

补录、录入和修改比分：

- 点击 `未赛` 色块，可以补录该场比分。
- 点击 `进行中` 色块，可以录入该场比分。
- 点击已完成比分色块，可以修改该场比分。
- 修改已完成比分前，需要输入确认密码 `789`。
- 弹窗会显示对战双方，操作方式和场地卡片里的比分录入保持一致。

排序方式：

- `初始排序`：按队伍原始顺序展示，不显示奖牌和前三名背景。
- `净胜分排名`：净胜分高者靠前。
- `胜场排名`：胜场高者靠前，胜场相同为相同排名。
- `胜场 > 净胜分`：先比胜场，再比净胜分。
- `胜场 > 净胜分 > 直接胜负`：胜场和净胜分都相同时，比较两队直接对战胜负。

点击 `导出 Excel` 可以导出当前排名对战表。
排序方式会自动保存，刷新页面后恢复上次选择。

## 6. 排位赛

### 6.1 从单循环排名生成

前置条件：

- 已经生成单循环赛程。
- 单循环所有比赛都已完成。
- 当前没有排位赛进度。

操作步骤：

1. 进入 `排位赛` 页签。
2. 在 `场地设置` 中设置排位赛 `场地数`。
3. 如需指定球馆实际场地，在场地编号输入区填写。
4. 晋级名额当前为 `前 4 名`。
5. 点击 `生成排位`。

对阵规则：

- 半决赛 1：第 1 名 vs 第 4 名。
- 半决赛 2：第 2 名 vs 第 3 名。
- 两场半决赛胜者进入冠军赛。
- 两场半决赛负者进入季军赛。

### 6.2 手动创建排位赛

如果不想依赖单循环结果：

1. 进入 `排位赛` 页签。
2. 在 `场地设置` 中设置排位赛 `场地数` 和场地编号。
3. 在 `手动创建` 区域输入 1 至 4 号种子队伍名称。
4. 点击 `创建排位`。

手动创建后，对阵仍按 1 vs 4、2 vs 3 生成。

### 6.3 排位赛拓扑图

排位赛创建后，页面会显示拓扑图：

- 左侧为半决赛。
- 右侧为冠军赛和季军赛。
- 已完成比赛显示比分。
- 点击未开始、进行中或已完成的比赛卡片，可以补录、录入或修改比分。
- 修改已完成排位赛比分前，需要输入确认密码 `789`。
- 如果冠军赛或季军赛已经有成绩，对应的半决赛成绩不能再修改。
- 每场比赛胜者会被标记。
- 冠军、亚军、季军会显示对应图标。

半决赛完成后：

- 胜者自动进入冠军赛。
- 负者自动进入季军赛。
- 冠军赛和季军赛由待定状态变为可安排状态。
- 如果修改半决赛比分导致胜负关系变化，冠军赛和季军赛会同步更新参赛队，受影响的后续场次需要重新安排和录入比分。
- 如果后续比赛已经录入成绩，需要先确认后续结果处理方式，再重建或清空排位赛进行调整。

### 6.4 排位赛安排比赛

排位赛使用和单循环相同的场地操作：

- `安排比赛`。
- `跳过一次`。
- `动态调配场地`。
- `撤回安排`。
- `记录比分并结束`。

排位赛数据和单循环数据独立。录入排位赛比分不会覆盖单循环比分。

### 6.5 清空排位赛

如需重建排位赛：

1. 在排位赛面板点击 `清空排位`。
2. 在弹窗中输入确认密码 `789`。
3. 点击确认后，排位赛进度被清空。
4. 单循环数据保留。

### 6.6 清空积分赛程

如需重建单循环赛程：

1. 在积分赛程配置条点击 `清空赛程`。
2. 在弹窗中输入确认密码 `789`。
3. 点击确认后，积分赛程和已录入的单循环进度被清空。
4. 排位赛数据不会被误清空。

## 7. 数据保存和恢复

系统会自动保存进度到当前浏览器的 `localStorage`。

会保存：

- 队伍数量和名称。
- 场地数量和场地编号。
- 单循环赛程和状态。
- 排位赛赛程和状态。
- 当前场地占用情况。
- 比分。
- 跳过记录。
- 动态调配推荐。
- 排位赛晋级名额和手动创建队伍名称。

还会单独保存界面偏好：

- 当前页签。
- 单循环赛程表的 `计划表` / `现场表` 选择。
- 排位赛赛程表的 `计划表` / `现场表` 选择。
- 排名对战表排序方式。

刷新页面后会自动恢复。

### 7.1 导出和导入数据

顶部右侧提供全局数据操作：

- `导出数据`：下载当前比赛进度和界面偏好的 JSON 文件。
- `导入数据`：选择 MatchFlow 导出的 JSON 文件，将其中数据写入当前浏览器。

导出时可以修改文件名，系统会自动补全 `.json` 后缀。

导入说明：

- 导入会覆盖当前本地比赛进度和界面偏好。
- 点击 `覆盖导入` 后，需要输入确认密码 `789`。
- 导入成功后页面会自动刷新，并恢复导入的数据。
- 该功能适合把比赛进度发给其他人，或在另一台电脑上继续操作。

注意：

- 数据只保存在当前浏览器和当前域名下。
- 换电脑、换浏览器、清除浏览器缓存后，数据不会自动保留，建议先使用 `导出数据` 备份。

## 8. 常见问题

### 8.1 为什么安排比赛按钮不能点？

通常是因为推荐比赛中的某支队伍还在其他场地进行中。可以等待上一场结束，或点击 `动态调配场地` 找一场当前可安排的比赛。

### 8.2 为什么赛程表显示调至其他场地？

说明这场比赛原计划在某个场地进行，但实际通过动态调配或现场安排到了另一个场地。

### 8.3 实际位置是什么意思？

系统会比较实际发生位置和计划位置：

- 小圆序号表示这场比赛在实际场地时间线中的第几场。
- 小圆序号颜色跟实际场地一致。

### 8.4 跳过一次会不会永久跳过这场比赛？

不会。跳过只是当前场地的推荐顺延。所有候选跳过一轮后，会重新从第一个候选开始。

### 8.5 清空单循环会不会清空排位赛？

不会。单循环和排位赛有各自的清空入口，互不覆盖。清空前都需要输入确认密码 `789`。

### 8.6 录入排位赛比分会不会影响单循环排名？

不会。排位赛比分独立保存，不参与单循环排名对战表。

### 8.7 为什么刷新后还停留在上次的页签或视图？

这是系统记住的界面偏好。当前页签、赛程表视图和排名排序方式都会自动恢复，方便继续现场操作。

### 8.8 忘记清空密码怎么办？

当前版本清空确认密码固定为 `789`。

## 9. 建议现场流程

1. 赛前配置队伍和场地编号。
2. 生成单循环赛程。
3. 导出计划表备用。
4. 各场地按推荐安排比赛。
5. 遇到场地快慢不一致时，使用 `动态调配场地`。
6. 比赛结束立即录入比分。
7. 单循环全部结束后检查排名对战表。
8. 需要排位赛时，从排名生成或手动创建。
9. 完成排位赛后查看拓扑图确认冠亚季军。
